In this video, I show you the new auto-export superpowers added in Obsidian Excalidraw 2.13.0!

Previously, the auto-export feature could export all your drawings, or export specific drawings based on frontmatter, but the exported images would always end up in the same folder as your source Excalidraw file. This was a problem many of you asked about for years, and I'm excited to share the solution available now.

With version 2.13.0, you can set up custom logic using JavaScript hooks to control exactly where your exported images (SVG, PNG, or even .excalidraw) are saved. This allows for incredibly fine-grained control, letting you automate your workflow based on criteria like frontmatter tags, file naming conventions, or even the folder structure of your source files.

I walk you through setting up the ExcalidrawStartup.md script, explain the data available in the onImageExportPath hook (like the file path, the action being performed - export, move, or delete), and show you how to implement different logic types. I highlight why using filename conventions in your logic might be better than frontmatter when it comes to automatically handling file deletions.

I also demonstrate a neat bonus feature: the ability to open the original Excalidraw drawing directly from the embedded exported image in your markdown note, even if the files are in different folders.

This update offers immense flexibility for automating your visual thinking workflow. While it requires some basic JavaScript understanding and thorough testing, it opens up powerful possibilities for organizing your assets.
